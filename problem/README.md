# problem/

本目录用于保存项目的外部问题定义、约束条件、评价指标、允许操作、禁止操作和提交要求。

这里记录的是“问题本身是什么”，不是“我们打算怎么优化”。

---

## 本目录应该存放什么

存放以下内容：

```text
problem/
├── README.md
├── technical-specification.md
├── scoring-and-sla.md
├── allowed-and-forbidden.md
└── submission-requirements.md
```

---

## 文件说明

| 文件                           | 作用                    |
| ---------------------------- | --------------------- |
| `technical-specification.md` | 原始任务说明、技术背景、目标系统、运行环境 |
| `scoring-and-sla.md`         | 评分规则、性能指标、达标条件、失败条件   |
| `allowed-and-forbidden.md`   | 允许做的优化、禁止做的修改、合规边界    |
| `submission-requirements.md` | 最终需要提交的源码、文档、脚本、环境说明等 |

---

## 内容边界

本目录只记录外部规则，不记录个人分析。

应该放入：

* 任务目标
* 运行环境
* 固定参数
* 测试方式
* 评分公式
* 性能指标定义
* 精度或正确性要求
* 禁止行为
* 提交材料要求
* 复现要求

不应该放入：

* 优化猜想
* 实验方案
* 代码改动
* 跑分结果
* 个人洞察
* 最终结论
* 临时讨论记录

---

## 记录格式

记录规则时，必须使用如下格式：

```md
## 规则编号
R001

## 规则内容
这里写清楚规则本身。

## 影响范围
说明它影响哪些操作，例如参数、代码、数据、接口、运行环境等。

## 风险等级
高 / 中 / 低

## 违反后果
说明违反该规则可能导致的结果，例如成绩无效、实验不可比较、结果不可复现等。

## 备注
补充解释。
```

---

## 维护要求

当外部规则发生变化时，应优先更新本目录。

任何优化方案在进入实验前，都应先检查：

* 是否违反固定环境要求
* 是否违反接口或参数约束
* 是否影响正确性或精度
* 是否绕过了正常评测流程
* 是否会造成结果不可复现
* 是否需要额外说明或声明

本目录是所有 Agent 和人工决策的规则来源。
