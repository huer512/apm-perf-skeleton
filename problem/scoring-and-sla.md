# 评分规则与 SLA

> 本文件由使用者或 Agent 从原始任务描述中抽取填写;填写前不视为已初始化。

## 核心指标定义
(逐个指标写清统计口径:p50/p99/均值、测量窗口、聚合方式、单位。指标定义不清会导致所有实验不可比较。)

## 评分公式 / 达标条件
(评分如何计算,达标线是什么。)

## 噪声阈值
(改进幅度低于该阈值时,必须重复实验确认稳定后才可登记为证据。
阈值应按本领域实际测量波动填写,不要凭空设定。)

## 验收标准

每条验收标准使用 AC 编号,同时写正例与负例;负例用于封堵取巧路径。

### AC-1 (标准名称)
- 内容:(可验证的验收描述)
- 正例(应判通过):(什么样的结果算满足)
- 负例(应判失败):(什么样的结果看似满足但必须判失败。
  例如:只修改负载定义使指标变好,应判失败;
  用 microbenchmark 结果冒充端到端结果,应判失败;
  牺牲正确性换来的性能提升,应判失败。)

## 失败条件
(什么情况下成绩无效,例如精度不达标、违反固定环境要求、结果不可复现。)
