# Experiment Directory Template

> **本目录为实验模板（E000），不参与实际任务记录，请勿修改。**  
> 创建新实验时，复制本目录结构，按 `E001_xxx`、`E002_xxx` 命名，从 **E001** 开始编号。

---

本目录用于保存一次独立实验的完整材料。

一个实验应该能够回答：

1. 本次实验想验证什么？
2. 它关联哪个假设？
3. 相比对照组改了什么？
4. 如何运行？
5. 得到了什么结果？
6. 结果是否可信？
7. 该实验支持还是反驳了原假设？
8. 下一步应该怎么做？

---

## 目录结构

```text
.
├── README.md
├── plan.md
├── remote_ref.yaml
├── run_commands.sh
├── code/
├── results/
├── logs/
├── analysis.md
└── conclusion.md
```

---

## 文件说明

| 文件或目录             | 说明                        |
| ----------------- | ------------------------- |
| `plan.md`         | 实验计划，运行前编写                |
| `remote_ref.yaml` | 通过 `server_id` 引用 `remote/servers.private.yaml`，并记录代码位置、commit、分支、产物路径等 |
| `run_commands.sh` | 实验运行命令                    |
| `code/`           | 代码补丁、改动说明、代码包             |
| `results/`        | 原始结果、解析结果、指标对比            |
| `logs/`           | 服务日志、压测日志、构建日志、错误日志       |
| `analysis.md`     | 结果分析，运行后编写                |
| `conclusion.md`   | 实验结论，分析后编写                |

---

## plan.md 应包含

```md
# 实验计划

## 实验编号
Exxx

## 实验名称
填写实验名称。

## 关联假设
Hxxx

## 实验目的
说明本实验要验证什么。

## 对照组
说明与哪个实验、版本或配置对比。

## 实验变量
说明本次实验改变了什么。

## 固定条件
说明哪些参数、环境、数据、接口或流程保持不变。

## 关注指标
- 指标 1
- 指标 2
- 指标 3

## 成功标准
说明什么结果算成功。

## 失败标准
说明什么结果算失败或无效。

## 风险
说明本实验可能带来的风险。

## 预期结果
说明实验前的预期。
```

---

## analysis.md 应包含

```md
# 实验分析

## 实验是否成功执行
说明实验是否完整跑通。

## 数据完整性检查
说明结果、日志、指标是否完整。

## 与对照组对比
说明主要指标变化。

## 性能变化分析
解释性能提升或下降的可能原因。

## 正确性或稳定性检查
说明是否存在正确性、精度、稳定性、失败率等问题。

## 异常现象
记录异常日志、异常指标或不符合预期的现象。

## 对假设的影响
说明本实验支持、削弱还是反驳了关联假设。
```

---

## conclusion.md 应包含

```md
# 实验结论

## 结论摘要
用几句话说明本实验结论。

## 是否有效
yes / no / partial / invalid

## 是否支持关联假设
supported / rejected / inconclusive

## 是否进入后续优化
yes / no / pending

## 关键证据
列出关键结果文件或日志位置。

## 新产生的洞察
说明是否产生了新的跨实验洞察。

## 后续动作
说明下一步要继续验证、合并、回滚还是放弃。
```

## 更新顺序

推荐按照以下顺序维护本目录：

```text
1. 创建实验目录
2. 编写 plan.md
3. 填写 remote_ref.yaml
4. 编写或复制 run_commands.sh
5. 执行实验
6. 保存 code/
7. 保存 results/
8. 保存 logs/
9. 编写 analysis.md
10. 编写 conclusion.md
11. 更新 experiments_index.md
12. 更新 memory/
```

---

## 注意事项

* 不要覆盖原始结果。
* 不要只保存截图，尽量保存机器可读结果。
* 不要只写结论，必须保留支持结论的证据。
* 无效实验也要记录，避免后续重复踩坑。
* 如果实验依赖远程服务器，必须在 `remote/servers.private.yaml` 中配置 SSH 信息，并在 `remote_ref.yaml` 中填写 `server_id`、远程路径、分支和 commit。
