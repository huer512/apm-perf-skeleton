# code/

本目录用于保存本次实验涉及的代码改动、补丁文件、代码包和改动说明。

如果被优化项目源码位于远程服务器，本目录不需要保存完整源码，只需要保存能够说明和复现本次改动的最小材料。

---

## 推荐内容

```text
code/
├── README.md
├── patch.diff
├── changed_files.md
├── code_notes.md
└── code_package.tar.gz
```

其中 `code_package.tar.gz` 是可选项，仅在确实需要保存完整代码包时使用。

---

## 文件说明

| 文件                    | 作用                    |
| --------------------- | --------------------- |
| `patch.diff`          | 本次实验相对对照组的代码差异        |
| `changed_files.md`    | 本次实验修改了哪些文件、每个文件的修改目的 |
| `code_notes.md`       | 补充说明代码实现思路、风险点、依赖关系   |
| `code_package.tar.gz` | 可选，保存完整代码包或关键源码快照     |

---

## changed_files.md 推荐格式

```md
# 修改文件列表

| 文件路径 | 修改内容 | 修改目的 | 风险等级 | 关联假设 |
|---|---|---|---|---|
| path/to/file | 简要说明 | 简要说明 | 高/中/低 | Hxxx |
```

---

## code_notes.md 推荐格式

```md
# 代码说明

## 修改摘要
说明本次实验的主要代码改动。

## 实现思路
说明为什么这样改。

## 关键路径
列出关键函数、模块或配置。

## 潜在风险
说明可能影响正确性、性能、稳定性或兼容性的地方。

## 回滚方式
说明如何恢复到对照组状态。
```

---

## 不建议保存的内容

不建议将以下内容放入本目录：

* 完整大型源码仓库
* 编译缓存
* 临时构建目录
* 二进制产物
* 大型第三方依赖包
* 密钥或私有配置
* 与本次实验无关的代码

---

## 维护要求

每次实验涉及代码改动时，应至少保存：

* `patch.diff`
* `changed_files.md`

如果没有代码改动，而只是配置、参数或运行方式变化，也应在 `code_notes.md` 中说明：

```md
本次实验无代码改动，仅修改运行配置。
```

本目录的目标是保证实验结果可以追溯到具体代码变化。
