# 排查决策树(triage tree)

> 由使用者按领域填写;填写前本文件不视为已初始化。
> 把"遇到某类症状先查什么、再查什么、什么情况换方法"写成可执行路径,
> 防止 Agent 凭症状直接改代码(证据驱动而非症状驱动)。

## 条目格式

```md
### 症状:(一类可观测问题,如"吞吐达标但尾延迟超标")

1. 先查:(检查项 + 命令/位置)
2. 再查:(检查项 + 命令/位置)
3. 候选方向:(按可能性排序的假设方向,可引用 references/heuristics.md 规则编号)
4. 换方法触发条件:(如"以上检查全部正常时,改用 X 手段深挖")
```
