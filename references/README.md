# references/

本目录是**领域先验知识库**:把"资深工程师如何判读数据、如何排查瓶颈"的经验写成
Agent 可套用的规则文件,由使用者按自己的领域填写。

与 memory/ 的区别:

```text
references/ = 领域先验(开工前就该知道的判读规则与排查路径,来自专家经验)
memory/     = 实测沉淀(本项目实验产生的洞察、证据、决策)
```

Agent 在分析结果与定位瓶颈时,应**先套用本目录的规则**;规则未覆盖时才可即兴判断,
且即兴判断必须标注置信度并注明"references 未覆盖"。实测反复验证的判读经验,
应从 memory/insight_bank.md 反哺回本目录。

## 文件

| 文件 | 作用 |
| --- | --- |
| `heuristics.md` | 判读规则:现象 → 解读 → 建议动作(含"能证明/不能证明"边界) |
| `triage-tree.md` | 排查决策树:症状 → 检查清单 → 候选方向 → 换方法触发条件 |

两个文件在填写前均不视为已初始化;骨架不预置任何领域内容。
