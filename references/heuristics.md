# 判读规则(heuristics)

> 由使用者按领域填写;填写前本文件不视为已初始化。
> 每条规则用封闭措辞写清"看到什么 → 说明什么 → 该做什么",并声明证明边界,
> 让 Agent 套用规则而不是即兴发挥。

## 规则格式

```md
### HEU-001 规则标题

- 现象:(可观测的指标/日志/曲线特征,写判定条件而非模糊描述)
- 解读:(这个现象说明什么;多个可能原因按可能性排序)
- 建议动作:(下一步查什么或改什么)
- 能证明:(该现象足以支持的结论)
- 不能证明:(常见的过度推断,禁止越界)
- 置信度:高 / 中 / 低(该规则本身的可靠程度)
```
